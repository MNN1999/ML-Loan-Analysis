Credit Policy Mirror

A loan portfolio of ~20k applications where a simple, interpretable model learns the bank's approval logic and then highlights the few decisions that don't match that logic.

The flow is straightforward:

  • Load loans into SQL and engineer risk & affordability bands.
  • Train logistic regression to mimic the bank's approval behaviour.
  • Compare model vs history to detect high-value anomalies for review.

Problem

A bank is approving and rejecting thousands of loans, but the true approval policy is hidden inside those decisions. Medium-risk customers are especially tricky: some are approved with heavy debt load, others are rejected despite looking affordable.

Analysis

We load the data into PostgreSQL on Neon, engineer risk, income and ticket-size bands in SQL, and build a logistic regression in Python that learns how the bank actually behaves. Then we check calibration and slice affordability by risk band.

‚úÖ

Solution

Use the trained model as a mirror of the policy: when the model is very confident but disagrees with the historical decision, that loan is flagged as an anomaly. The output is a short, actionable review list for risk or underwriting teams.
